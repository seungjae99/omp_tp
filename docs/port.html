Project Context

03 핵심 구현 상세
Obstacle Detection — 카메라 이진화 감지

깊이 센서 없이 카메라 단일 센서로 장애물을 판정합니다. 장애물(어두운 물체)이 화면에 등장하면 어두운 픽셀 수가 임계치를 초과하는 원리를 이용합니다. 

def _on_image(self, msg: Image):
    cv_img = self.bridge.imgmsg_to_cv2(msg, desired_encoding='bgr8')
    gray   = cv2.cvtColor(cv_img, cv2.COLOR_BGR2GRAY)

    # dark pixel extraction: threshold=50, inverted binary
    _, bw  = cv2.threshold(gray, 50, 255, cv2.THRESH_BINARY_INV)
    count  = cv2.countNonZero(bw)

    self.obstacle = (count > 10_000)   # pixel threshold

    # publish grayscale for rviz2 debug
    gray_msg = self.bridge.cv2_to_imgmsg(gray, encoding='mono8')
    gray_msg.header = msg.header
    self.pub_gray.publish(gray_msg)
설계 의도: 그레이스케일 임계치 50은 시뮬레이션 환경 조명 기준으로 설정. 픽셀 임계치 10,000은 노이즈 오감지를 줄이기 위해 obs_avoid_1의 8,000에서 상향 조정.

Avoidance Sequence — 7-Step S-Curve Path

좌회전으로 이탈 → 장애물 측면 통과 → 우회전으로 복귀하는 대칭 구조. 각 스텝은 타이머 경과(elapsed ≥ duration)로 자동 전환됩니다.

Control Loop — Timer-based Execution
def _control_loop(self):
    now     = self.get_clock().now()
    elapsed = (now - self.state_start).nanoseconds / 1e9

    if self.state == State.NAVIGATING:
        if self.obstacle:
            self.state       = State.AVOIDING
            self.avoid_step  = 0
            self.state_start = now
        else:
            self._drive(self.FWD_SPEED, self.FWD_SPEED)

    else:  # AVOIDING
        action, duration = self.avoid_sequence[self.avoid_step]
        if   action == "forward": self._drive( self.FWD_SPEED,  self.FWD_SPEED)
        elif action == "left":    self._drive(-self.TURN_SPEED, self.TURN_SPEED)
        elif action == "right":   self._drive( self.TURN_SPEED, -self.TURN_SPEED)

        if elapsed >= duration:
            self.avoid_step += 1
            self.state_start  = now
            if self.avoid_step >= len(self.avoid_sequence):
                self.state = State.NAVIGATING
참고: elapsed 계산은 nanosecond 단위로 수행 후 초 단위로 환산. state_start를 스텝 전환 시마다 갱신하여 각 단계 타이머가 독립적으로 동작합니다.

Known Limitations
한계 1: 카메라 단일 센서 의존 — 조명 변화나 장애물 색상에 따라 감지 임계치 재조정 필요
한계 2: 고정 시간 기반 시퀀스 — 장애물 크기·거리에 무관하게 동일한 경로 실행 (피드백 없음)
한계 3: 회피 중 새로운 장애물 감지 시 현재 시퀀스 완료 후 재감지 — 복합 장애물 환경 미지원
